# Canonical Daily Skills

These skills should remain small, stable, and predictable.

## Recommended canonical set
- `feature-planning`
- `tdd-workflow`
- `systematic-debugging`
- `code-review`
- `verification-before-completion`
- `refactoring`
- `security-review`
- `performance-audit`
- `api-design` (optional but recommended)

## Rule
One capability = one canonical skill.
If duplicates exist, keep one canonical and move others to `skills/internal/`.
