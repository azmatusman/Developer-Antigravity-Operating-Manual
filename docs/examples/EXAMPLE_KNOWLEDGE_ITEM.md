# Example: Knowledge Item

## When to create
Create a KI only if it is durable:
- decision
- constraint
- root cause
- invariant
- rejected option with rationale

## Example
**Title:** "Strategies emit signals only"
- Durable Statement: Strategies produce signals; execution layer produces orders.
- Why: prevents coupling; improves testing.
- Evidence: tests; review notes.

Use `/new-knowledge-item` template.
