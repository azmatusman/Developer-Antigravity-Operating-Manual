# /plan

**Purpose:** Create a structured implementation plan before coding.

## Required behavior
- Use Sequential Thinking to propose **2–3 architectures** with tradeoffs.
- Recommend **ONE**.
- Produce:
  - `implementation_plan.md`
  - `task_checklist.md`
- **STOP** and wait for approval before coding.

## Best practice
- Keep plan tasks atomic.
- Identify parallel tasks explicitly.
- Include testing strategy.
