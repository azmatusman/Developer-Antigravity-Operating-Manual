# Session Management

## Where brainstorming fits
Use brainstorming only when requirements are unclear, risky, or broad.
- Brainstorming happens **before `/plan`**.
- Do not brainstorm after architecture approval.

## SUMMARY.md vs Knowledge Items

### SUMMARY.md (short-term continuity)
Use at end of every meaningful session.
- Intent
- Completed
- Decisions
- Open items
- Next steps
- Artifacts

### Knowledge Items (long-term memory)
Create only for:
- non-obvious decisions
- constraints
- root causes
- invariants
- tradeoffs/rejected options

## When to start a new chat
Start a new chat when intent changes:
- planning → implementation
- implementation → debugging
- switching subsystem
- conversation becomes long/noisy

Rule: **one chat = one intent**.
