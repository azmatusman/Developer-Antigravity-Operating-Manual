# Sprint Planning Protocol

1. List the features/tasks for this sprint
2. For each task, invoke the **feature-planning** skill (brief mode):
   - One-paragraph scope
   - Files affected
   - Dependencies on other tasks
   - Complexity estimate
3. Order tasks by dependencies (independent tasks first)
4. Identify which tasks can run in PARALLEL via Agent Manager
5. Generate a sprint board:

| # | Task | Complexity | Dependencies | Status | Agent |
|---|------|-----------|-------------|--------|-------|

6. **STOP** — Wait for approval before execution
7. On approval → execute tasks in order, using /tdd for each
