# Documentation Protocol

1. Analyze the current codebase structure
2. Check existing documentation for gaps or outdated content
3. Generate/update the requested documentation type:

   **If README**: Project overview, setup instructions, usage, architecture summary
   **If API docs**: All endpoints with request/response schemas and examples
   **If Architecture**: System diagram (text-based), module responsibilities, data flow
   **If Changelog**: Categorized list of changes (Added, Changed, Fixed, Removed)
   **If Onboarding**: Setup steps, project structure guide, key concepts, common tasks

4. Use clear, concise language
5. Include examples where helpful
6. **STOP** — Present for review before committing
